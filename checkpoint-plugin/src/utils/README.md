# Agent Session Raw Collector

`collect_sessions.py` records raw local agent session artifact paths plus the raw agent settings paths that define the environment around those sessions. It does not normalize messages or transform data.

Run:

```bash
python3 collect_sessions.py
```

Default output goes to `checkpoint/raw_sessions/<timestamp>/` with a `manifest.json`. By default the script only records native source paths, file sizes, and session context references; it does not copy large transcripts, logs, caches, or runtime databases.

To copy raw files as well:

```bash
python3 collect_sessions.py --copy
```

It uses the agents' native locations and records settings in `sessionContextRefs`. Provider-wide settings are linked to every referenced session. Project-scoped settings are linked to the sessions whose raw transcript exposes a `cwd`.

The table below is the collection checklist. Paths that exist and are readable are recorded in the manifest and are copied only when `--copy` is used. OS keychains, server-side policy, and MDM/enterprise policy that is not materialized as a local file are documented here for completeness, but cannot be collected by this script.

## Codex paths

| Path | Data |
| --- | --- |
| `~/.codex/sessions/**/*.jsonl` | Active session transcripts and run events generated while Codex runs. |
| `~/.codex/archived_sessions/**/*.jsonl` | Archived session transcripts and run events. |
| `~/.codex/session_index.jsonl` | Session index metadata, such as thread ids, names, and update times. |
| `~/.codex/history.jsonl` | Prompt history when history persistence is enabled. |
| `$CODEX_HOME/config.toml` or `~/.codex/config.toml` | User config: model, model provider, approval policy, sandbox, permissions, MCP servers, tools, features, hooks, telemetry, shell environment policy, profiles, custom agents, skills enablement, and related runtime defaults. |
| `$CODEX_HOME/*.config.toml` or `~/.codex/*.config.toml` | Named profile configs selected with `--profile`. |
| `/etc/codex/config.toml`, `/etc/codex/requirements.toml`, `/etc/codex/skills/**` | System config, admin-enforced requirements, and admin skills on Unix-style systems. |
| `/Library/Managed Preferences/com.openai.codex.plist` | macOS managed preferences if enterprise policy is written to disk. |
| `~/.codex/mcp.json` | Legacy or external MCP server config if present. Current Codex commonly stores MCP under `config.toml` as `mcp_servers.*`. |
| `~/.codex/auth.json` | File-based auth/session data when Codex is not using the OS keychain/keyring. |
| `~/.codex/AGENTS.md`, `~/.codex/AGENTS.override.md` | User/global instructions loaded before project instructions. |
| `~/.agents/skills/**`, `~/.codex/skills/**`, `~/.codex/plugins/**`, `~/.codex/vendor_imports/**` | User skills, Codex/system skill cache when present, plugins, and imported/bundled skill metadata. |
| `~/.codex/agents/*.toml` | User custom subagent definitions. |
| `~/.codex/hooks.json` | User lifecycle hooks when configured outside `config.toml`. |
| `~/.codex/rules/**` | User command/permission rules. |
| `~/.codex/models_cache.json`, `~/.codex/model-catalogs/**` | Model catalog/cache data when present. |
| `~/.codex/log/**`, `~/.codex/*.sqlite*`, `~/.codex/sqlite/**`, `~/.codex/goals_*.sqlite*`, `~/.codex/logs_*.sqlite*`, `~/.codex/state_*.sqlite*` | Runtime-generated logs, resumable state, goals, telemetry/local state databases, and sidecar WAL/SHM files. |
| `~/.codex/memories/**`, `~/.codex/memories_extensions/**` | Generated or persisted memory data. |
| `~/.codex/shell_snapshots/**` | Runtime shell state snapshots. |
| `~/.codex/app-server-daemon/**`, `~/.codex/app-server-control/**`, `~/.codex/node_repl/**`, `~/.codex/computer-use/**` | Runtime state generated by Codex app/server, Node REPL, and computer-use integrations. |
| `~/.codex/cache/**`, `~/.codex/packages/**` | Runtime caches, downloaded packages, and bundled runtime artifacts. |
| `<repo>/.codex/config.toml`, `<repo>/.codex/requirements.toml` | Project config and requirements: trusted project overrides for model, approvals, sandbox, permissions, MCP, skills, hooks, rules, features, and subagent defaults. |
| `<repo>/.codex/hooks.json` | Project lifecycle hooks. |
| `<repo>/.codex/rules/**`, `<repo>/.codex/agents/*.toml`, `<repo>/.agents/skills/**`, `<repo>/.codex/skills/**` | Project command/permission rules, custom subagents, official project skills, and legacy/local project skill trees when present. |
| `<repo>/AGENTS.md`, `<repo>/AGENTS.override.md` | Project instructions; Codex walks from project root to session `cwd` and loads matching files. |
| OS keychain/keyring, server-side account policy, CLI flags, `--config` overrides, live `/permissions` or runtime overrides | Auth tokens, managed settings, and per-run overrides may live outside readable files. These are not copied unless the agent writes a local file listed above. |

## Claude paths

| Path | Data |
| --- | --- |
| `~/.claude/projects/**/*.jsonl` | Session transcripts, tool calls/results, model usage, cwd, permission mode, attachments, MCP/tool/skill listing deltas, and other run events. |
| `~/.claude/sessions/**/*.jsonl` | Additional session transcript storage used by some Claude Code versions or installs. |
| `~/.claude/settings.json` | User settings: model, permissions, hooks, env vars, plugins, auto memory, tool behavior, sandboxing, status line, output style, subagent selection, and other defaults. |
| `~/.claude/config.json`, `~/.claude/claude.json` | Additional user-level Claude Code config/state when present. |
| `~/.claude.json` | OAuth/session data, user and local MCP servers, per-project state, trust/allowed tools, and caches. |
| `/Library/Application Support/ClaudeCode/managed-settings.json`, `/Library/Application Support/ClaudeCode/managed-settings.d/*.json`, `/Library/Application Support/ClaudeCode/managed-mcp.json`, `/etc/claude-code/managed-settings.json`, `/etc/claude-code/managed-settings.d/*.json`, `/etc/claude-code/managed-mcp.json` | File-based managed settings, managed setting fragments, and managed MCP config on macOS and Unix-style systems. |
| `/Library/Managed Preferences/com.anthropic.claudecode.plist` | macOS managed preferences if enterprise policy is written to disk. |
| `~/.claude/CLAUDE.md` | User/global memory and instructions. |
| `~/.claude/rules/**` | User rules loaded with Claude memory/instructions. |
| `~/.claude/skills/**` | User skills. |
| `~/.claude/agents/**` | User subagents. |
| `~/.claude/commands/**` | User slash commands. |
| `~/.claude/output-styles/**` | User output style definitions. |
| `~/.claude/projects/<project>/memory/**` | Auto memory for the project associated with a session. |
| `~/.claude/file-history/**` | Runtime-generated file history snapshots referenced by sessions. |
| `~/.claude/cache/**`, `~/.claude/backups/**`, `~/.claude/downloads/**`, `~/.claude/paste-cache/**`, `~/.claude/plans/**`, `~/.claude/session-env/**` | Runtime-generated caches, backups, downloads, paste cache, plans, and session environment snapshots. |
| `~/.claude/shell-snapshots/**`, `~/.claude/tasks/**`, `~/.claude/todos/**`, `~/.claude/telemetry/**`, `~/.claude/statsig/**`, `~/.claude/ide/**` | Runtime-generated shell snapshots, task/todo state, local telemetry/feature flag state, and IDE integration state. |
| `~/.claude/plugins/**` | Installed/plugin marketplace data and enablement metadata that can provide skills, agents, hooks, and MCP servers. |
| `<repo>/.claude/settings.json` | Shared project settings: permissions, hooks, env, MCP/plugin enablement, model-related settings, status line, output style, and tool behavior. |
| `<repo>/.claude/settings.local.json` | Local project settings; personal overrides and permissions not checked into git. |
| `<repo>/.mcp.json` | Shared project MCP server definitions. |
| `<repo>/CLAUDE.md`, `<repo>/.claude/CLAUDE.md`, `<repo>/CLAUDE.local.md` | Project and local memory/instructions loaded for sessions under that repo. |
| `<repo>/.claude/rules/**` | Project rules, including path-scoped rules. |
| `<repo>/.claude/skills/**` | Project skills. |
| `<repo>/.claude/agents/**` | Project subagents. |
| `<repo>/.claude/commands/**` | Project slash commands. |
| `<repo>/.claude/output-styles/**` | Project output style definitions. |
| Server-managed settings, Windows registry policy, OS keychain/keyring, server-side account policy, CLI flags, `--permission-mode`, runtime `/model` or `/config` changes, MDM policy not written to disk | Auth tokens, entitlements, managed settings, and per-run overrides may live outside readable files. These are not copied unless the agent writes a local file listed above. |
