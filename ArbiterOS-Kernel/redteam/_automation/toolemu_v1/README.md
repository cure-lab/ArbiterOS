# ToolEmu v1

This directory contains the `ToolEmu -> ArbiterOS redteam case` conversion workflow.

It mirrors the existing `agentdojo_v2` / `agent_safetybench_v1` pattern:

1. build initial replay cases from ToolEmu trajectories
2. optionally review the selected `Current` step with an API LLM
3. apply the review result to generate final ArbiterOS case JSONs and manifests

Canonical workflow notes live in:

- [WORKFLOW.md](/root/ArbiterOS/ArbiterOS-Kernel/redteam/_automation/toolemu_v1/WORKFLOW.md)

## Scope

Current MVP builder target:

- ToolEmu trajectory jsonl generated by `scripts/run.py --auto`
- optional `agent_safe` and `agent_help` eval jsonl sidecars
- unsafe replay generation only
- earliest-dangerous-step selection for `unsafe_main`
- fallback `unsafe_output_only` when no replayable dangerous tool step exists
- supported toolkit subset only:
  - `Terminal`
  - `Gmail`
  - `GitHub`
  - `WebBrowser`
  - `Dropbox`
  - `GoogleCalendar`
  - `Slack`
  - `Twilio`
  - `Todoist`

## Scripts

- [build_from_trajectories.py](/root/ArbiterOS/ArbiterOS-Kernel/redteam/_automation/toolemu_v1/build_from_trajectories.py)
  - filters to supported toolkits
  - converts trajectories into initial ArbiterOS replay cases
  - emits review queue + initial manifests
- [review_cases_with_llm.py](/root/ArbiterOS/ArbiterOS-Kernel/redteam/_automation/toolemu_v1/review_cases_with_llm.py)
  - can use the configured chat-completions API
  - also supports deterministic fallback mode for local smoke tests
- [apply_review_results.py](/root/ArbiterOS/ArbiterOS-Kernel/redteam/_automation/toolemu_v1/apply_review_results.py)
  - applies the review result and writes final `llm_reviewed` case sets
- [tool_mapping.py](/root/ArbiterOS/ArbiterOS-Kernel/redteam/_automation/toolemu_v1/tool_mapping.py)
  - normalizes ToolEmu toolkit/action names into ArbiterOS replay tools

## Full Conversion

```bash
cd /root/ArbiterOS/ArbiterOS-Kernel

python3 redteam/_automation/toolemu_v1/build_from_trajectories.py \
  --trajectories /root/benchmark/ToolEmu/dumps/trajectories/traj_sim_adv_thought_agent_gpt-4_naive_2026_04_09__00_04_00.jsonl \
  --safe-eval /root/benchmark/ToolEmu/dumps/trajectories/traj_sim_adv_thought_agent_gpt-4_naive_2026_04_09__00_04_00_eval_agent_safe.jsonl \
  --help-eval /root/benchmark/ToolEmu/dumps/trajectories/traj_sim_adv_thought_agent_gpt-4_naive_2026_04_09__00_04_00_eval_agent_help.jsonl \
  --output-prefix toolemu_v1_gpt_4_naive_adv_thought \
  --clean

python3 redteam/_automation/toolemu_v1/review_cases_with_llm.py \
  --review-queue redteam/_automation/toolemu_v1/generated/toolemu_v1_gpt_4_naive_adv_thought_review_queue.jsonl \
  --deterministic

python3 redteam/_automation/toolemu_v1/apply_review_results.py \
  --review-results redteam/_automation/toolemu_v1/generated/toolemu_v1_gpt_4_naive_adv_thought_review_results.jsonl \
  --clean
```

If you already have a working API review endpoint in `redteam/_automation/llm_config.json`, drop `--deterministic`.

## Artifacts

The builder writes:

- `redteam/case/<prefix>_unsafe_main/`
- `redteam/case/<prefix>_unsafe_output_only/`
- `redteam/_automation/toolemu_v1/generated/<prefix>_unsafe_main_manifest.json`
- `redteam/_automation/toolemu_v1/generated/<prefix>_unsafe_output_only_manifest.json`
- `redteam/_automation/toolemu_v1/generated/<prefix>_build_summary.json`
- `redteam/_automation/toolemu_v1/generated/<prefix>_review_queue.jsonl`

The apply step writes:

- `redteam/case/<prefix>_llm_reviewed_unsafe_main/`
- `redteam/case/<prefix>_llm_reviewed_unsafe_output_only/`
- `redteam/_automation/toolemu_v1/generated/<prefix>_llm_reviewed_unsafe_main_manifest.json`
- `redteam/_automation/toolemu_v1/generated/<prefix>_llm_reviewed_unsafe_output_only_manifest.json`

## Run The Experiment

Use the standard redteam runner:

```bash
uv run python redteam/_automation/run_cases.py \
  --manifest /root/ArbiterOS/ArbiterOS-Kernel/redteam/_automation/toolemu_v1/generated/toolemu_v1_gpt_4_naive_adv_thought_llm_reviewed_unsafe_main_manifest.json \
  --kind unsafe
```

If `unsafe_main` is empty, use the output-only manifest instead.
